Onboard to a project

Get your footing in a real open-source project before you change anything.
Learning outcome
You can navigate a real open-source project — its purpose, people, and contribution workflow — well enough to make a first move.

The activity

Pick (or get assigned) an active open-source project and learn how it actually works: what it does, who maintains it, how contributions flow, and where the community talks.

How to do it

  1. Read the README, CONTRIBUTING guide, and code of conduct.
  2. Set the project up locally and get it running.
  3. Find where the community lives — issues, discussions, chat — and how decisions get made.
  4. Trace the contribution workflow end to end: fork → branch → pull request → review.
What to show
A short orientation note: what the project does, how to contribute, and one thing that surprised you.
